= 了解ONTAP Select資料保護和效率
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
ONTAP Select基於ONTAP儲存軟體構建，透過高可用性、無共享的橫向擴展架構高效地提供企業儲存服務。您可以部署包含一個、兩個、四個、六個或八個節點的解決方案，每個節點的 NFS、SMB/CIFS 和 iSCSI 連接儲存的原始容量高達 400 TB。您可以利用原生重複資料刪除和壓縮功能來增加有效容量，從而降低儲存成本。橫向擴展架構支援高可用性和無中斷資料移動，以實現負載平衡或維護硬體。



== 快照資料保護

ONTAP Select包含資料保護功能，包括快照和SnapMirror軟體。您可以快速將資料複製到其他ONTAP存儲，無論是在本地端、遠端站點還是在雲端。如果您必須快速恢復數據， SnapRestore軟體可以使用本機快照在幾秒鐘內恢復整個檔案系統或資料卷，無論檔案容量或數量如何。



== MetroCluster軟體定義存儲

ONTAP Select MetroCluster軟體定義儲存 (SDS) 提供增強的保護和經濟高效的實施。

如果滿足某些最低要求，雙節點叢集可以在兩個位置之間擴展。此架構完美地融合了基於硬體的MetroCluster和單一資料中心叢集（硬體定義或軟體定義）。 ONTAPONTAP Select MetroCluster SDS 的要求凸顯了軟體定義儲存解決方案的普遍靈活性，以及它與基於硬體的MetroCluster SDS 之間的差異。無需專有硬體。

與MetroCluster不同， ONTAP Select使用現有的網路基礎架構，並支援高達 5 毫秒 RTT 的網路延遲和高達 5 毫秒的最大抖動，總最大延遲為 10 毫秒。最大距離也要求為 10 公里，儘管延遲情況更為重要。市場空間中的分離要求與物理分離有關，而不是實際距離。在某些情況下，這可能意味著不同的建築物。在其他情況下，它可能意味著同一建築物中的不同房間。無論實際的實體位置為何，雙節點叢集定義為MetroCluster SDS 的原因是每個節點都使用單獨的上行鏈路交換器。

作為雙節點 HA 配置的一部分，需要一個調解器在故障轉移期間正確識別活動節點，並避免任何裂腦情況（即兩個節點在網路分區期間保持獨立活動狀態）。此操作與先前提供的常規雙節點 HA 配置相同。為了在站點故障期間提供適當的保護和故障轉移，調解器應與兩個 HA 節點位於不同的站點。調解器與每個ONTAP Select節點之間的最大延遲不能超過 125 毫秒。

ONTAP Select MetroCluster SDS 具有以下優勢：

* MetroCluster SDS 為ONTAP Select提供了另一個維度（資料中心到資料中心）的保護。現在，除了利用軟體定義儲存和ONTAP的所有優勢之外，您還可以利用此額外的保護等級。
* MetroCluster SDS 提供業務關鍵型資料保護，具有 0 RPO 和自動故障轉移功能。資料儲存和應用程式存取點均可自動切換到正常運作的資料中心或節點，無需 IT 人員幹預。
* MetroCluster SDS 經濟高效。它利用現有的網路基礎架構，在 HA 對之間實現擴展彈性，無需額外硬體。它還能在同一叢集中提供主動/主動資料存取和資料中心冗餘。
